# 806.Java Concurrency API中的Lock接⼝(Lock interface)是什么？对⽐同步它有什么优势？

> 原文：[https://zwmst.com/4869.html](https://zwmst.com/4869.html)

   [ *并发编程面试专栏* ](https://zwmst.com/%e5%b9%b6%e5%8f%91%e7%bc%96%e7%a8%8b%e9%9d%a2%e8%af%95%e4%b8%93%e6%a0%8f)*[ <time datetime="2021-10-13T00:06:26+08:00"> 2021-10-12 </time> ](https://zwmst.com/4869.html)  1.  Lock接⼝⽐同步⽅法和同步块提供了更具扩展性的锁操作。他们允许更灵活的结构，可以具有完全不同的性质，并且可以⽀持多个相关类的条件对象。
2.  它的优势有：
    1.  可以使锁更公平
    2.  可以使线程在等待锁的时候响应中断
    3.  可以让线程尝试获取锁，并在⽆法获取锁的时候⽴即返回或者等待⼀段时间
    4.  可以在不同的范围，以不同的顺序获取和释放锁*
